## 记录

### [#001] 2026-06-08 — 定位重构：从模糊到清晰
- **用户意图**: 评审发现"定位不清晰，和 Learning System 关系模糊"，用户不知道什么时候该用 Codex 而不是 LS
- **调用**: 全系统审视 + 竞品对比分析
- **输入**: "重写 README，明确 Codex 不是 LS 的替代品或竞品，而是不同场景下的互补工具"
- **输出**:
  - 完整的中文 README 重写：
    - 首屏核心声明："Codex Learning Skills = 你的每日学习教练，不是学习系统的替代品"
    - 「LS vs Codex 对比表」：
      - LS：系统化学习设计、长周期课程规划、知识管道搭建
      - Codex：即时问答解惑、碎片时间学习、考试冲刺辅助
    - 使用场景矩阵：5 个典型场景 × 2 个系统的推荐度
    - 11 个技能分组展示（记忆/理解/应用/分析/评价/创造 + 元认知技能）
- **效果**: ✅ 完全解决
- **耗时**: 约 30 min
- **改进点**: 对比表应该加入真实用户案例（如："小明用 LS 规划了 3 个月的机器学习课程，每天用 Codex 解惑具体概念"），目前只有理论对比

### [#002] 2026-06-08 — 产品化落地：从文档到教练
- **用户意图**: 明确"不是替代品是每日教练"的定位后，需要让 README 从技术文档变成可感知的产品
- **调用**: 用户体验设计 + 信息架构优化
- **输入**: "README 首屏要让用户在 10 秒内知道'这是什么'和'我什么时候该用它'"
- **输出**:
  - 首屏三段式结构：
    1. **一句话定位**："把学习科学变成随叫随到的私人教练"
    2. **核心差异**："LS 帮你设计学习系统，Codex 帮你在系统中高效执行"
    3. **30 秒傻瓜式入口**：3 个按钮 → "我是备考党" / "我想提升理解力" / "我需要记忆方法"
  - 11 技能卡片式展示：每个技能配 icon + 一句话说明 + 典型使用场景
  - 新增「谁该用 Codex」章节：目标用户画像 × 3（学生/职场人/终身学习者）
- **效果**: ✅ 完全解决
- **耗时**: 约 20 min
- **改进点**: 缺少"试用体验"环节——用户无法在阅读 README 后立即体验某个 skill，建议嵌入交互式 demo 或示例对话

## 模式分析

**什么情况下用得最好：**
- **定位澄清任务**（#001）：当系统边界模糊时，通过对比矩阵和使用场景能快速理清关系
- **产品化改造任务**（#002）：从用户视角出发，用"10 秒法则"强制精简信息架构

**什么情况下容易失败：**
- 暂无明确失败模式（当前样本量仅 2 条）
- 潜在风险：如果未来新增 skill 导致系统膨胀，可能再次出现定位漂移

**下一步优化方向：**
1. **真实案例库**：收集 5-10 个"LS + Codex 联合使用"的真实故事，嵌入 README 作为 social proof
2. **交互式入口**：在 README 中提供可直接复制的 prompt 模板，让用户零门槛试用
3. **智能路由原型**：探索"用户描述需求 → 自动推荐用 LS 还是 Codex"的路由逻辑
4. **定期定位审计**：每季度检查一次与 LS/AIGC/LinguaFlow 的边界是否清晰
